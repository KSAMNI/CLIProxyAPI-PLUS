# Debugging (disciplined, no rushing)

Mandatory process when investigating a bug:
1. Reproduce the symptom and environment first.
2. Trace the request or event path step by step.
3. Verify state at each boundary before changing code.
4. Narrow the suspect area with file:line or log evidence.
5. Write the failing test or reproducible check first.
6. Fix minimally within scope.

> Never guess and fix on the spot before you have evidence.
